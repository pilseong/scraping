REST 서비스를 구현하기 위해서 1-1 Spring MVC -> 고전적인 방식 Front Controller 모델이다. 1-2 Spring WebFlux -> Reactive 방식으로 동작하는 REST를 구현하는 기술이다. 1-3 Spring WebFlux fn -> Reactive 방식으로 동작하는데 functonal programming model을 사용한다. 1-3-1 Node.js의 express 서버를 자바로 구현한 거라 생각하면 된다. 2. REST 서비스를 사용하기 위한 클라이언트로 2-1 Spring RestTemplate -> 고전적인 방식 2-2 Spring WebClient -> 스프링의 reactive 클라이언트를 구현하는 기술이다. 솔직히 좋은지는 모르겠다. 위의 내용만 이해하면 Spring REST는 끝난다. 그런데, 스프링은 무슨 슬라임 같아서 손에 잡히는 것이 잘 없다. 그래서 하나하나 분해해서 학습하고 구현하고 정리해야지 어느 정도 머리에 남게 된다.
